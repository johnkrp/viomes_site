# VIOMES Homepage Memory

This file stores the rules and facts that future agents and users should keep in mind while working in this repo.

## Always Follow

- Keep changes minimal and aligned with the existing React + TypeScript + Vite stack.
- Update the core docs in the same change set when an important project change lands.
- Prefer the existing project-context docs and workflow notes over creating new parallel docs.
- Do not remove active catalog assets, generated JSON, or project-context documents without checking references first.
- After any design changes, run `npm run build` and check the output using Playwright for unexpected changes before deploying.

## Stable Project Facts

- The catalog pipeline is workbook -> `src/data/generate_catalog_json.py` -> generated JSON in `src/data` and `public/data` -> `src/lib/catalogDataLoader.ts` -> page components.
- Routing and route-specific SEO live in [src/App.tsx](src/App.tsx).
- App bootstrap and theme setup live in [src/main.tsx](src/main.tsx).
- Catalog data loads from `public/data` first and falls back to bundled `src/data` JSON.
- Temporary test packshots are controlled by `VITE_USE_TEST_PACKSHOTS=true` in [src/lib/testPackshotOverrides.ts](src/lib/testPackshotOverrides.ts).
- Long-form project notes belong in `project-context/1.define/`, `project-context/2.build/`, or `project-context/3.deliver/`.
- Visual design rules live in [DESIGN.md](DESIGN.md).
- The live site is hosted on Plesk at https://test.viomes.gr.
- Deploys are generated by running `npm run build` and uploading the resulting `dist` folder to the Plesk server.

## Documentation Rule

If a change affects routes, catalog data flow, page structure, scripts, workflow docs, or cleanup guidance, update this file, [PROJECT_OVERVIEW.md](PROJECT_OVERVIEW.md), [README.md](README.md), and [DESIGN.md](DESIGN.md) together.

## Cleanup Guidance

- Treat build output, dependency folders, caches, and generated screenshots as disposable cleanup targets.
- Treat repo-local agent files as disposable only if they reappear.
- Keep `project-context`, `plans/README.md`, and the root overview docs unless they are truly obsolete.
