# VIOMES Homepage Memory

This file stores the rules and facts that future agents and users should keep in mind while working in this repo.

## Always Follow

- Keep changes minimal and aligned with the existing React + TypeScript + Vite stack.
- Update the core docs in the same change set when an important project change lands.
- Prefer the existing project-context docs and workflow notes over creating new parallel docs.
- Do not remove active catalog assets, generated JSON, or project-context documents without checking references first.
- After any design changes check the output using Playwright for unexpected changes.

## Stable Project Facts

- The catalog pipeline is workbook -> `src/data/generate_catalog_json.py` -> generated JSON in `src/data` and `public/data` -> `src/lib/catalogDataLoader.ts` -> page components.
- Routing and route-specific SEO live in [src/App.tsx](src/App.tsx).
- App bootstrap and theme setup live in [src/main.tsx](src/main.tsx).
- Catalog data loads from `public/data` first and falls back to bundled `src/data` JSON.
- Temporary test packshots are controlled by `VITE_USE_TEST_PACKSHOTS=true` in [src/lib/testPackshotOverrides.ts](src/lib/testPackshotOverrides.ts).
- The homepage VERDIA/GUSTO brand strip lives in [src/components/home/BrandVerdiaSection.tsx](src/components/home/BrandVerdiaSection.tsx). Treat it as an art-directed composition, not a normal fluid text section.
- Long-form project notes belong in `project-context/1.define/`, `project-context/2.build/`, or `project-context/3.deliver/`.
- Visual design rules live in [DESIGN.md](DESIGN.md).
- The live site is hosted on Plesk at https://test.viomes.gr.
- Deploys are generated by running `npm run build` and uploading the resulting `dist` folder to the Plesk server.
- Graphify outputs live in `graphify-out/` and are part of the architecture workflow:
  - `GRAPH_REPORT.md` (summary), `graph.html` (visual map), `graph.json` (queryable graph).

## Brand Section Rules

- For `BrandVerdiaSection`, use a fixed-aspect desktop canvas so the composition scales as one poster-like unit.
- Do not let the desktop headline rely on natural wrapping. Lock deliberate line breaks with block spans and `whitespace-nowrap` so wide screens do not change the typographic rhythm.
- Avoid independently responsive section heights for this brand strip. Percent positioning only stays reliable when the parent canvas aspect ratio is stable.
- If the desktop strip must be full-bleed, remove max-width caps on the canvas rather than adding side padding. If it must be capped, accept intentional side margins.
- Mobile should use a separate simplified stacked layout instead of squeezing the desktop poster into a narrow viewport.
- Keep brand-section image crop adjustments in frame size, `object-fit`, and `object-position`; avoid runtime image-ratio state unless a real content requirement appears.

## Graphify Operating Routine

Command cheat sheet: [docs/GRAPHIFY_WORKFLOW.md](docs/GRAPHIFY_WORKFLOW.md)

- During an active chat, do not refresh after every code edit.
- Refresh graph structure at the start of a new chat session (or when explicitly requested):
  - `.\.venv-graphify\Scripts\graphify update .`
- For full semantic refresh (docs/images/papers + code), run:
  - `$graphify .` in Codex chat
- For architecture investigation, prefer:
  - `graphify query`, `graphify path`, `graphify explain`
    over ad-hoc broad search when relationship tracing is needed.
